## 1. Problèmes UX par criticité

| ID | Criticité | Constat | Statut |
|----|-----------|---------|--------|
| U1 | 🔴 | **Hors-ligne : rien.** SPA GitHub Pages sans service worker → sans réseau, l'app ne charge pas du tout. Incompatible avec l'usage soirée (tablette, wifi incertain) et la philosophie local-first. | ✅ **Corrigé dans cette passe** : PWA (vite-plugin-pwa/Workbox), précache intégral (29 entrées, 1,65 Mo, bouteilles incluses). Vérifié : SW `activated`, 14 webp + fond + index servables depuis le cache. |
| U2 | 🟠 | **Non installable** : pas de manifest → pas d'icône sur l'écran d'accueil de la tablette/téléphone du barman. | ✅ **Corrigé** : manifest complet (nom, icônes 192/512 + maskable, standalone, thème `#0a0705`). |
| U3 | 🟢 | Métadonnées documentaires absentes (`description`, `theme-color`) : partage de lien pauvre, barre navigateur grise. | ✅ **Corrigé** dans `index.html`. |
| U4 | 🟢 | Bouteilles ajoutées à la main : pas de champ image dans le formulaire (fallback pastille prévu). | ⏳ Backlog — champ photo optionnel si le besoin d'autonomie se confirme. |
| U5 | 🟢 | Données localStorage mono-appareil (PC ≠ tablette) : contournement = export/import JSON via Gestion. | Assumé (décision actée, Supabase seulement si besoin réel). |

**Vérifié sans constat (mesures du 2026-07-08, viewport 375px, données réelles injectées)** :
navigation 4 onglets + badge OK ; cibles tactiles ≥ 44px sur Inventaire/Commande/Soirée (seul
« élément <44px » = input fichier volontairement masqué, sa cible = le label) ; aucun débordement
horizontal ; formulaires tous labellisés ; états vides tous couverts (cave, commande, soirée,
aucun résultat) ; erreurs → toasts (y compris échec d'écriture localStorage).

## 2. Incohérences visuelles

Rien de nouveau depuis les audits précédents. Le point connu (« grammaire mixte » : tuiles
ambiance Sommelier ≠ pills kiosque ≠ lignes Inventaire) est requalifié : ce sont des **variantes
par contexte désormais documentées** dans MASTER.md §5 (tuile = exploration posée, pill 52px =
kiosque debout, ligne dense = console barman). L'incohérence résiduelle serait à traiter par le
refactor design-system (§4), pas par des retouches ponctuelles.

## 21st.dev

**Aucun composant remplacé.** Critère du brief appliqué : aucun composant existant n'est
« réellement faible ou incohérent » au point de justifier un import externe — tous sont
fonctionnels, testés et cohérents avec l'identité artisanale. Importer des composants génériques
créerait l'incohérence que ce brief cherche à éviter.
